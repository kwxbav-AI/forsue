# 確認「內容篇數扣工時」是否有從總工時扣除 — 詳細步驟

當你懷疑某一天某門市（例如 2/13 東勇）的總工時沒有扣到內容篇數的 2 小時時，請依下列步驟操作。

---

## 前置：確認專案有在跑

1. 在專案目錄打開終端機，執行：
   ```bash
   npm run dev
   ```
2. 看到類似 `Local: http://localhost:3000` 即表示成功。
3. 以下步驟都假設網址是 **http://localhost:3000**，若你的 port 不同請自行替換。

---

## 步驟一：用除錯 API 看「當日」資料

用來確認：  
- 當天有沒有「東勇」的內容篇數資料、扣幾小時  
- 系統有沒有把扣工時算到東勇門市  
- 目前 DB 裡東勇的總工時是多少、和預期是否一致  

### 1-1 在瀏覽器開除錯網址

1. 打開瀏覽器（Chrome / Edge 等）。
2. 在網址列輸入（日期請改成你要查的那天）：
   ```
   http://localhost:3000/api/performance/debug-deduction?date=2026-02-13
   ```
3. 按 Enter，畫面會顯示一串 JSON。

### 1-2 看回傳內容在說什麼

回傳大致會長這樣（僅示意）：

```json
{
  "date": "2026-02-13",
  "contentEntries": [
    { "branch": "東勇", "deductedMinutes": 120, "deductedHours": 2 }
  ],
  "contentDeductionHoursByStore": {
    "某storeId": { "storeName": "東勇", "hours": 2 }
  },
  "perStore": [
    {
      "storeName": "東勇",
      "rawHours": 10,
      "deductionHours": 2,
      "expectedTotalHours": 8,
      "actualTotalHours": 8,
      "match": true
    }
  ]
}
```

請對照下面檢查：

| 項目 | 要看什麼 | 代表什麼 |
|------|----------|----------|
| **contentEntries** | 有沒有一筆 `branch: "東勇"`、`deductedMinutes: 120`（或 deductedHours: 2） | 2/13 是否有東勇的內容篇數填報且扣 2 小時 |
| **contentDeductionHoursByStore** | 有沒有一筆對應「東勇」、`hours: 2` | 系統有沒有把這 2 小時算到東勇門市 |
| **perStore** 裡「東勇」那一筆 | **rawHours**、**deductionHours**、**expectedTotalHours**、**actualTotalHours**、**match** | 原始工時、扣工時、預期總工時、目前 DB 總工時、是否一致 |

- 若 **contentEntries** 沒有東勇或沒有 2 小時 → 請到「內容篇數填報」檢查 2/13 是否真的有東勇、扣 2 小時的資料。
- 若 **contentEntries** 有東勇 2 小時，但 **perStore** 裡東勇的 **deductionHours** 是 **0** → 代表「分店」名稱和門市名稱對不起來（例如 DB 門市是「東勇店」、填報是「東勇」），需要讓兩邊名稱一致或請開發調整對應邏輯。
- 若 **deductionHours** 是 2，但 **actualTotalHours** 沒有少 2、或 **match** 是 **false** → 代表目前 DB 的總工時還沒被重算過，需要做「步驟二」重算。

---

## 步驟二：重算該日（或區間）績效

扣工時是在「重算每日績效」時寫入總工時的，所以若步驟一發現應該要扣但沒扣，就要再重算一次。

### 方法 A：只重算「單日」（例如 2/13）

1. 在「每日工效比」或任一有登入系統的頁面，按 **F12** 打開開發者工具。
2. 切到 **Console（主控台）** 分頁。
3. 貼上下面這行（日期可改），按 Enter：
   ```javascript
   fetch('/api/performance/recalculate-daily', {
     method: 'POST',
     headers: { 'Content-Type': 'application/json' },
     body: JSON.stringify({ date: '2026-02-13' })
   }).then(r => r.json()).then(console.log);
   ```
4. 若成功，主控台會顯示類似：`{ success: true, date: "2026-02-13" }`。

### 方法 B：重算「一段日期」（例如 2/1～2/28）

1. 同樣 F12 → Console。
2. 貼上（起日、迄日可改），按 Enter：
   ```javascript
   fetch('/api/performance/recalculate-daily', {
     method: 'POST',
     headers: { 'Content-Type': 'application/json' },
     body: JSON.stringify({ startDate: '2026-02-01', endDate: '2026-02-28' })
   }).then(r => r.json()).then(console.log);
   ```
3. 成功會顯示：`{ success: true, startDate: "2026-02-01", endDate: "2026-02-28" }`。

### 方法 C：重算「全部歷史」

1. 在系統中進入 **目標設定** 頁（或設定裡有「重算」的頁面）。
2. 按 **「重算全部歷史績效」** 按鈕並確認。
3. 會呼叫 `POST /api/performance/recalculate-all`，重算所有有資料的日期。

---

## 步驟三：重算後再查一次除錯 API

重算完成後，再確認一次「預期」和「實際」是否一致。

1. 再次打開：
   ```
   http://localhost:3000/api/performance/debug-deduction?date=2026-02-13
   ```
2. 看 **perStore** 裡「東勇」：
   - **expectedTotalHours** = rawHours − 2（扣 2 小時）
   - **actualTotalHours** 應該要等於 **expectedTotalHours**
   - **match** 應該為 **true**

若 **match** 為 true，就代表 2/13 東勇的總工時已正確扣掉內容篇數的 2 小時。

---

## 步驟四：在畫面上確認「每日工效比」

1. 打開 **每日工效比** 頁面。
2. 日期選 **2026-02-13**。
3. 找到 **東勇** 那一列，看 **總工時** 是否為你預期的數字（例如原本 10 小時、扣 2 小時後應為 8 小時）。

---

## 常見狀況整理

| 狀況 | 可能原因 | 建議做法 |
|------|----------|----------|
| contentEntries 沒有東勇 | 該日沒有東勇的內容篇數資料，或日期選錯 | 到「內容篇數填報」確認 2/13 是否有東勇、扣 2 小時 |
| 有東勇 2 小時但 deductionHours 是 0 | 分店名稱與門市名稱不一致 | 對照 API 回傳的 storeName 與 contentEntries 的 branch，讓名稱一致或調整對應邏輯 |
| deductionHours 是 2 但 actualTotalHours 沒少 2 | 尚未重算該日 | 依步驟二執行「重算單日」或「重算區間」 |
| 重算後仍不對 | 門市名稱對應或程式邏輯問題 | 把 debug-deduction 的完整 JSON 提供給開發人員排查 |

---

## 若要用 Postman / 其他工具

- **除錯（查詢）**  
  - 方法：**GET**  
  - 網址：`http://localhost:3000/api/performance/debug-deduction?date=2026-02-13`

- **重算單日**  
  - 方法：**POST**  
  - 網址：`http://localhost:3000/api/performance/recalculate-daily`  
  - Body：JSON `{ "date": "2026-02-13" }`

- **重算區間**  
  - 方法：**POST**  
  - 網址：`http://localhost:3000/api/performance/recalculate-daily`  
  - Body：JSON `{ "startDate": "2026-02-01", "endDate": "2026-02-28" }`

以上步驟做完，就可以完整確認「內容篇數填報（含扣工時）」的 2 小時是否有在總工時中扣除，以及如何修正。
