# 對外網址部署步驟（讓同事用網址就能開）

要讓程式有**固定網址**、不在公司也能用，需要兩件事：**雲端資料庫** + **雲端主機**。以下用 **Neon（免費 PostgreSQL）** + **Vercel（免費 Next.js 託管）** 為例。

---

## 一、準備雲端資料庫（Neon）

1. 打開 [neon.tech](https://neon.tech)，用 Google/GitHub 註冊登入。
2. 建立新專案（Project），選區域（例如 Singapore）。
3. 建立完成後，在專案裡會看到 **Connection string**，格式類似：
   ```text
   postgresql://使用者:密碼@ep-xxx.region.aws.neon.tech/資料庫名?sslmode=require
   ```
4. 複製這串網址，等一下要給 Vercel 用。

---

## 二、本機先測試雲端資料庫（建議）

1. 在專案資料夾複製一份環境變數：
   ```bash
   cp .env .env.backup
   ```
2. 編輯 `.env`，把 `DATABASE_URL` 改成 Neon 的連線字串。
3. 執行：
   ```bash
   npx prisma migrate deploy
   npx prisma db seed
   ```
4. 執行 `npm run dev`，在瀏覽器開 http://localhost:3000 確認功能正常。
5. 沒問題後，可把 `.env` 改回本機資料庫（或保留，之後部署用同一份）。

---

## 三、程式碼放到 GitHub

1. 若還沒用 Git，在專案資料夾執行：
   ```bash
   git init
   ```
2. 新增 `.gitignore`（若沒有），確保有：
   ```text
   .env
   node_modules
   .next
   ```
3. 到 [github.com](https://github.com) 登入，新增一個 **New repository**（例如 `daily-performance-system`），不要勾選 README 等。
4. 在本機專案資料夾執行（請把 `你的帳號`、`daily-performance-system` 換成你的）：
   ```bash
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/你的帳號/daily-performance-system.git
   git branch -M main
   git push -u origin main
   ```
5. 重新整理 GitHub 頁面，確認程式碼已上傳。

---

## 四、用 Vercel 部署（得到對外網址）

1. 打開 [vercel.com](https://vercel.com)，用 **GitHub** 登入。
2. 點 **Add New… → Project**，從列表選你剛 push 的 **daily-performance-system**，按 **Import**。
3. **Environment Variables** 區塊：
   - Name：`DATABASE_URL`
   - Value：貼上 Neon 的連線字串（第一節複製的那串）
   - 環境選 **Production**（或三個都勾）。
4. 按 **Deploy**，等一兩分鐘。
5. 完成後會出現 **Visit** 按鈕，點下去就是你的對外網址，例如：
   ```text
   https://daily-performance-system-xxx.vercel.app
   ```
6. 把這個網址傳給同事，他們用瀏覽器開就能用（不需同網路）。

---

## 五、之後程式有改動

1. 本機改完後執行：
   ```bash
   git add .
   git commit -m "說明你的修改"
   git push
   ```
2. Vercel 會自動偵測 GitHub 的 push，重新部署，幾分鐘後網址上的版本就會更新。

---

## 六、若不想用 Neon（其他雲端資料庫）

- **Supabase**： [supabase.com](https://supabase.com) → 建立專案 → Settings → Database → Connection string（選 URI）。
- **Railway**： [railway.app](https://railway.app) → New Project → PostgreSQL → 建立後在 Variables 看 `DATABASE_URL`。

把取得的 `DATABASE_URL` 貼到 Vercel 的環境變數即可，其餘步驟同上。

---

## 常見問題

| 問題 | 處理方式 |
|------|----------|
| 部署後開網址出現 500 / 資料庫錯誤 | 到 Vercel 專案 → Settings → Environment Variables，確認 `DATABASE_URL` 正確且已重新部署（Deployments → 最新一筆 → Redeploy）。 |
| 想用自己的網域 | Vercel 專案 → Settings → Domains，新增你的網域並依指示設定 DNS。 |
| 資料庫連線數不夠 | Neon 免費方案有連線數上限；若流量大可升級或換其他雲端 DB。 |

完成上述步驟後，你就會有一個**對外網址**，同事隨時用瀏覽器開啟即可使用。
