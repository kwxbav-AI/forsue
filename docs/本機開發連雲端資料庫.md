# 本機開發連到雲端資料庫

> **GitHub 共編、團隊多人開發**：建議改用 [團隊共用開發資料庫.md](./團隊共用開發資料庫.md)（Neon 獨立 dev 庫 + 1Password 分享連線，不 commit `.env`）。

本機 `http://localhost:3000` 若 **營收報表沒資料**、**Dashboard 顯示尚無營收或出勤**，但正式網站正常，幾乎都是因為：
- 本機 `.env` 的 `DATABASE_URL` 指向 **本機 PostgreSQL**（空的或只有測試資料）
- 正式站 Vercel 指向 **Neon 等雲端資料庫**（已上傳 2026/4 營收與出勤）

營收報表、圖表、Dashboard **共用同一資料庫**，改連線後三處會一併正常。

---

## 步驟一：確認目前連到哪裡

在專案資料夾執行：

```bash
npm run db:check
```

- 若顯示 **本機資料庫** 且營收筆數為 0 → 請繼續步驟二。
- 若顯示 **雲端／遠端** 且筆數 > 0 → 重啟 `npm run dev` 後再開報表即可。

開發中也可在瀏覽器開（僅本機、非 production）：

`http://localhost:3000/api/dev/db-status`

---

## 步驟二：取得 DATABASE_URL

**團隊開發**：向管理員索取「開發用」連線（見 [團隊共用開發資料庫.md](./團隊共用開發資料庫.md)），不必從 Vercel 複製。

**個人／對正式站同一套數字**：任選一種方式：
### A. 從 Vercel 後台複製（最簡單）

1. 登入 [vercel.com](https://vercel.com) → 你的專案
2. **Settings** → **Environment Variables**
3. 複製 **DATABASE_URL** 的完整值（Production 或 Preview 皆可，建議與正式站相同）

### B. 用 Vercel CLI 拉下環境變數

```bash
npm i -g vercel
vercel login
vercel link
vercel env pull .env.local
```

之後 Prisma / Next 會讀取 `.env.local`（若與 `.env` 並存，請確認最終生效的是雲端那串）。

---

## 步驟三：寫入本機環境變數

1. 開啟專案根目錄的 **`.env`**（沒有就從 `.env.example` 複製）
2. 將 `DATABASE_URL` **整行替換**為雲端連線字串，例如：

   ```env
   DATABASE_URL="postgresql://user:pass@ep-xxxx.neon.tech/neondb?sslmode=require"
   ```

3. **不要**把含密碼的 `.env` commit 到 Git（已在 `.gitignore`）

---

## 步驟四：套用 schema 並重啟

```bash
npx prisma generate
npx prisma migrate deploy
npm run db:check
npm run dev
```

`db:check` 應顯示雲端主機名稱，且 **RevenueRecord、AttendanceRecord 筆數 > 0**。

## 常見問題

| 狀況 | 處理 |
|------|------|
| `db:check` 連線失敗 | 檢查 URL 是否完整、Neon 是否允許你目前 IP、是否加 `?sslmode=require` |
| 仍連 localhost | 確認改的是正在使用的 `.env`，且已 **關閉再開** `npm run dev` |
| 想保留本機 DB 做測試 | 可另建 `.env.local` 指雲端、`.env` 指本機；開發時以 `.env.local` 為準 |
| 只有營收沒有出勤 | Dashboard 工時仍會是 0，需兩種上傳都有 |

---

## 安全提醒

- 雲端 `DATABASE_URL` 等同正式資料庫權限，本機開發請勿對外分享 `.env`。
- 若僅需「看資料」、不要誤改正式資料，可向 DBA 申請 **唯讀** 連線字串（若有的話）。
